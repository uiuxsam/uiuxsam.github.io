DC HealthLink Website


UX Designing is highly depended on business requirements and user research, since I don’t have

those available, I assumed the current website as the requirement for content strategy.

Navigation Bars

1. One horizontal primary/parent navigation bar on the home page

2. Additional navigation bar may appear as a secondary/child navigation bar

3. Present these elements as un-clickable text content

Buttons

4. Button color to be red since red appears in the logo and can be used to maintain brand color

5. The red buttons are working like ‘Register” button. However, this button is not required

here since there is already a link on the top right corner to create an account. Therefore, it should say ‘View Plan’.

6. Clicking on “View Plan” button should take the user to a page where there would be

details on the plan available

Links

7. Remove the redundant links on the same page

8. Text links should be marked as visited once the user visits it

Text Content

9. Reduce the amount of text and use less colorful images

10. Highlight the notice on the top of the page with a bright background color

11. Customer service number may appear at the bottom of the page where usually look for addresses and hours. It may also appear on the top right side of the page depending on the business need and the overall design of the site

Look and feel

12. Home page feels overwhelming

13. Background image blending with front images makes the page look busy

Fonts

14. Number of font sizes used in a page should be limited unless it has data visualization graphs

15. Number of different font colors should be limited

Color Scheme

16. The color scheme should be improved with appropriate brand colors

17. Color contrast should be maintained all over the website

ANALYTICS


I have applied design review process to analyze this website. A design review is a usability-inspection method in which (usually) one reviewer examines a design to identify usability problems.

There are many design issues with this website. Some of the generic issues are -

Navigation Bars

1. There are three different types of horizontal navigation bars in one page

2. The navigation bars seem like parent or child to one another but it is not

3. Icons with text in a horizontal bar look like a secondary navigation bar which also look clickable links but they are not. It fails to meet consistency with User Expectations.

Buttons

4. Red buttons are mainly used for negative actions unless red is a brand color

5. The buttons on the page says ‘Get Started” which is very generic and the user does not know what to expect

6. After clicking “Get Started” button, the user ultimately lands on a page later on that requires creating an account

Links

7. There are redundant links on the same page. When the same links appear several times on the same page, users get confused by thinking the links have different contents

8. There is no way to understand if a link has already been visited

Text Content

9. Too much text and non-text content on the home page that is causing visual noise

10. The notice on the top of the page looks like other plain texts on the page which may cause the user ignoring the message

11. Customer service number appears twice on the same page which may make the user think those are different numbers

Look and feel

12. Home page feels overwhelming

13. Background image blending with front images makes the page look busy

Fonts

14. Seven different font sizes have been used on this page

15. Five different font colors have been used on this page

Color Scheme

16. The color scheme makes the site look inconsistent

17. Color contrast is not well maintained with different elements
